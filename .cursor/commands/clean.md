# Clean (Repository)

## Overview

This command runs a periodic repository hygiene check: identify essential vs disposable files, clean temporary and context storage, and organize execution folders so the structure stays predictable and maintainable. Use it weekly or when the repo feels cluttered.

**Usage**: `/clean` or "run repo clean"

**Note**: For context-only cleanup (mcp/terminal files older than 7 days), use `/context clean`. This command covers the full repo (context, .tmp, execution folders, untracked review).

---

## What Is Essential vs Disposable

Aligned with **AGENTS.md → File Organization**:

| Category | Location | Action |
|----------|----------|--------|
| **Essential** | `vaults/<area>/projects/<project>/plans/` | Keep. Do not delete. |
| **Essential** | `vaults/<area>/projects/<project>/tasks/` | Keep. Do not delete. |
| **Essential** | `vaults/<area>/projects/<project>/execution/*.py`, `*.sh`, `*.js` (reusable scripts) | Keep. Document in plans if new. |
| **Essential** | `vaults/<area>/projects/<project>/project/` (deliverables) | Keep. Prefer cloud outputs; local only if needed. |
| **Essential** | `.cursor/commands/`, `.cursor/rules/`, `.cursor/hooks/` | Keep. |
| **Essential** | `AGENTS.md`, `.env` (no secrets in chat) | Keep. |
| **Disposable** | `.tmp/` | Safe to delete. Regeneratable. |
| **Disposable** | `.mypy_cache/`, `.ruff_cache/`, `.pytest_cache/` | Safe to delete. Regeneratable. |
| **Disposable** | `context/mcp/*.md`, `context/terminal/*.txt` (old) | Clean if older than 7 days (see `/context clean`). |
| **Review** | `execution/*.md`, one-off scripts or outputs | Move to `project/` or `project/archive/` if worth keeping; else remove after task is done. |
| **Review** | Untracked files (`git status`) | Decide: add to repo, move to project/archive, or delete. |

**Rule of thumb**: If it was generated by a script for a single task, it is either intermediate (→ `.tmp/` or delete) or a one-off artifact (→ `project/` or `project/archive/`). Reusable scripts stay in `execution/`.

## Process / Workflow

Run these steps in order. Use dry-run or list-before-delete where indicated.

### Step 1: Context and .tmp status

1. Check context directories:
   ```bash
   du -sh context/mcp context/history context/terminal 2>/dev/null
   ```
2. Check `.tmp/` size and file count:
   ```bash
   du -sh .tmp 2>/dev/null; find .tmp -type f 2>/dev/null | wc -l
   ```
3. If context is large or `.tmp` has many files, note for cleanup in Step 2.

### Step 2: Safe cleanup (with confirmation)

1. **.tmp**: List contents; confirm with user; then:
   ```bash
   rm -rf .tmp/*
   ```
2. **Context**: Suggest `/context clean` (removes files in `context/mcp` and `context/terminal` older than 7 days). Do not delete `context/history/` unless user explicitly asks.

### Step 3: Execution folder hygiene

For each `vaults/**/execution/` directory that has many files:

1. List files and classify:
   - **Keep in execution**: Scripts used by plans or run repeatedly.
   - **Candidates to move or remove**: One-off outputs, one-off scripts, or large JSON/CSV that belong in `project/` or `.tmp/`.
2. Suggest moving one-off artifacts to `project/archive/` (create `archive/` if needed).
3. Do not delete or move files without user confirmation.

### Step 4: Untracked and uncommitted review

1. Run `git status`.
2. **Root hygiene**: If `.mypy_cache`, `.ruff_cache`, `.pytest_cache` appear in the repo root, confirm they are in `.gitignore`.
3. List untracked files and important modified files.
4. For each untracked path, suggest:
   - **Track**: Add to repo if it is a plan, task, or reusable script.
   - **Archive**: Move to `project/` and optionally add to `.gitignore`.
   - **Ignore**: Add to `.gitignore` if always local/temporary.
   - **Delete**: If clearly temporary and safe to remove.

Do not run `git add` or `git commit` unless the user explicitly asks.

### Step 5: Consistency check (optional)

1. Check that each project with `execution/` has a `plans/` or `tasks/` reference.
2. Report oddities (e.g. execution folders with only one-off outputs and no reusable scripts).

## Quick Reference

| Step | Action |
|------|--------|
| 1 | Context + .tmp status |
| 2 | Clean .tmp and/or context (with confirmation) |
| 3 | Execution hygiene: classify files, suggest move/archive/delete |
| 4 | Git status; suggest track/archive/ignore for untracked |
| 5 | Optional: plans/tasks/execution consistency |

- **Essential**: plans, tasks, reusable execution scripts, project deliverables, .cursor, AGENTS.md, .env.
- **Disposable**: .tmp/*, old context/mcp and context/terminal.
- **Review**: one-off .md/.py in execution, untracked files.

**Reference**: AGENTS.md → File Organization, Directory Reference. Context-only: `/context status`, `/context clean`.
